Redis慢查询功能

一、生命周期

慢查询发生在第三个阶段

客户端超时不一定是慢查询，但慢查询是客户端超时的一个可能因素

二、两个配置

1、slowlog-max-len

Redis慢查询是一个先进先出的队列

假如一条命令，在第三步的执行过程中，被列入慢查询的范围内，那么这条命令就会进入一个队列

# 单位是微秒
# 当一个命令的运行时间达到10毫秒后
# 就会被放进慢查询队列
slowlog-log-slower-than=10000

这个队列其实就是通过Redis列表来实现的

这个列表或者说队列，通常被称之为slowlog list

队列成员包含四个属性

  • id
  • time
  • duartion
  • command + 参数

这个队列的长度是固定的

# 设置慢查询队列长度为100
slowlog-max-len=100

当队列满了之后，最先进队列的会被最先踢出去

队列是保存在内存当中的

慢查询队列会随着Redis的重启而被重置

2、slowlog-log-slower-than

慢查询阀值，单位是微秒

slowlog-log-slower-than=0, 记录所有命令

slowlog-log-slower-than<0, 不纪录任何命令

3、配置方法

# 获取慢查询队列长度，默认值100
config get slowlog-max-len
# 获取慢查询阀值, 默认值10毫秒
config get slowlog-log-slower-than
# 修改配置文件重启
# 动态配置
config set slowlog-max-len 1000
config set slowlog-log-slower-than 1000s

三、三个命令

# 获取慢查询队列
slowlog get [n]
n是条数，n不能超过slowlog-max-len
# 获取慢查询队列长度
slowlog len
# 清空慢查询队列
slowlog reset

四、运维经验

1、slowlog-max-len不要设置过大，默认10ms，通常设置1ms

2、slowlog-log-slower-than不要设置过小，通常设置为1000左右
